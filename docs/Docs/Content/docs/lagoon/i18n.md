# Internationalisation (i18n)

Lagoon includes a full i18n framework for building multi-language documentation sites. It supports locale-aware URL routing, automatic UI string translation for 8 languages, right-to-left (RTL) layout, and a built-in language picker component.

## Quick start

Add a `Locales` dictionary to your `DocsConfig`. The key `"root"` designates the default locale (no URL prefix). All other keys become URL prefixes (e.g. `/fr/`, `/de/`).

```csharp
var config = new DocsConfig
{
    Title = "My Docs",
    Locales = new Dictionary<string, LocaleConfig>
    {
        ["root"] = new LocaleConfig { Label = "English", Lang = "en" },
        ["fr"]   = new LocaleConfig { Label = "Français", Lang = "fr" },
        ["de"]   = new LocaleConfig { Label = "Deutsch", Lang = "de" },
    },
};
```

With this configuration:

- English pages are served at `/docs/getting-started`
- French pages are served at `/docs/fr/getting-started`
- German pages are served at `/docs/de/getting-started`
- The `LanguagePicker` dropdown appears in the header automatically

## Content organisation

Locale-specific content lives in subfolders matching the locale key:

```
Content/
├── docs/
│   ├── getting-started.md        ← root (English)
│   ├── guides/
│   │   └── intro.md
│   ├── fr/
│   │   ├── getting-started.md    ← French
│   │   └── guides/
│   │       └── intro.md
│   └── de/
│       └── getting-started.md    ← German
```

When a page has not been translated, Lagoon falls back to the root locale content and displays an untranslated-content notice banner.

## LocaleConfig

Each locale is configured with a `LocaleConfig` record.

| Property | Type | Default | Description |
|---|---|---|---|
| `Label` | `string` | *(required)* | Display name shown in the language picker (e.g. "English", "Français") |
| `Lang` | `string` | *(required)* | BCP-47 language tag (e.g. `"en"`, `"fr"`, `"zh-CN"`). Used for the HTML `lang` attribute |
| `Dir` | `string` | `"ltr"` | Text direction: `"ltr"` or `"rtl"` |
| `Translations` | `UiTranslations` | Auto-resolved | Per-locale UI string overrides. When omitted, Lagoon auto-resolves from `BuiltInTranslations` |

### RTL support

For right-to-left languages, set `Dir = "rtl"`:

```csharp
["ar"] = new LocaleConfig { Label = "العربية", Lang = "ar", Dir = "rtl" },
```

Lagoon sets the HTML `dir` attribute and applies RTL-aware CSS layout.

## UI translations

All user-facing strings in Lagoon components are translatable. The `UiTranslations` record contains 21 string properties covering every UI element:

| Category | Properties |
|---|---|
| Skip link | `SkipLinkLabel` |
| Search | `SearchLabel`, `SearchPlaceholder`, `SearchDialogLabel`, `SearchCloseLabel`, `SearchResultsLabel`, `SearchNoResults` |
| Theme | `ThemeToggleLabel`, `ThemeSwitchToLight`, `ThemeSwitchToDark` |
| Navigation | `SidebarNavLabel`, `SiteNavigationLabel`, `MobileNavOpenLabel`, `BreadcrumbsLabel` |
| Pagination | `PaginationLabel`, `PaginationPrevious`, `PaginationNext` |
| Table of contents | `TocLabel` |
| Footer | `BuiltWithLabel` |
| Language picker | `LanguageSelectLabel` |
| Content | `UntranslatedContentNotice` |

### Built-in languages

Lagoon ships with complete translations for 8 languages via `BuiltInTranslations`:

| Language | Tag | Property | RTL |
|---|---|---|---|
| English | `en` | `BuiltInTranslations.English` | No |
| French | `fr` | `BuiltInTranslations.French` | No |
| Spanish | `es` | `BuiltInTranslations.Spanish` | No |
| German | `de` | `BuiltInTranslations.German` | No |
| Japanese | `ja` | `BuiltInTranslations.Japanese` | No |
| Arabic | `ar` | `BuiltInTranslations.Arabic` | Yes |
| Chinese (Simplified) | `zh-CN` | `BuiltInTranslations.ChineseSimplified` | No |
| Portuguese (Brazil) | `pt-BR` | `BuiltInTranslations.Portuguese` | No |

### Automatic resolution

When `LocaleConfig.Translations` is not explicitly set, Lagoon automatically resolves the correct built-in translation using `BuiltInTranslations.ForLanguage(lang)`. This method is case-insensitive and supports subtag fallback — for example, `"fr-CA"` resolves to French translations.

This means the quick-start example above automatically uses French and German UI strings without any extra configuration.

### Custom translations

Override individual strings by providing a custom `UiTranslations` instance:

```csharp
["fr"] = new LocaleConfig
{
    Label = "Français",
    Lang = "fr",
    Translations = new UiTranslations
    {
        SearchPlaceholder = "Chercher dans la doc...",
        BuiltWithLabel = "Propulsé par",
        // All other properties fall back to their English defaults.
        // To start from the French base, use BuiltInTranslations.French
        // and override with a `with` expression:
    },
},
```

To override just one or two strings while keeping the rest of a built-in translation:

```csharp
["fr"] = new LocaleConfig
{
    Label = "Français",
    Lang = "fr",
    Translations = BuiltInTranslations.French with
    {
        SearchPlaceholder = "Chercher dans la doc...",
    },
},
```

## Locale resolver

`LocaleResolver.Resolve()` maps a URL path to a `ResolvedLocale` record containing the matched locale key, configuration, path prefix, and remaining content path.

```csharp
var resolved = LocaleResolver.Resolve("/docs/fr/guides/intro", locales, basePath: "/docs");

// resolved.Key         → "fr"
// resolved.Config.Lang → "fr"
// resolved.PathPrefix  → "/fr"
// resolved.ContentPath → "/guides/intro"
```

Resolution order:

1. Strip the base path from the URL
2. Try to match a non-root locale prefix (`/fr/`, `/de/`, etc.)
3. Fall back to the `"root"` locale if no prefix matches
4. Fall back to the first locale in the dictionary if no `"root"` key exists

Returns `null` when no locales are configured (single-language mode).

## Path helpers

### LocalePathHelper

Static methods for locale-aware URL path manipulation.

| Method | Description |
|---|---|
| `PrefixPath(contentPath, localePrefix, basePath)` | Composes `basePath + localePrefix + contentPath` (e.g. `"/docs" + "/fr" + "/intro"` → `"/docs/fr/intro"`) |
| `StripPrefix(fullPath, localePrefix, basePath)` | Removes the base path and locale prefix, returning just the content path |
| `BelongsToLocale(path, localePrefix, basePath)` | Returns `true` if the path belongs to the given locale |

All methods have overloads that omit `basePath` (defaults to `""`).

### LocaleContentResolver

Static methods for mapping between content slugs and locale-scoped content paths.

| Method | Description |
|---|---|
| `GetLocaleContentPath(contentPath, localeKey)` | Prepends the locale key to a content path (e.g. `"guides/intro"` + `"fr"` → `"fr/guides/intro"`). Returns the path unchanged for the `"root"` locale |
| `StripLocaleFromSlug(slug, localeKey)` | Removes the locale prefix from a content slug (e.g. `"fr/guides/intro"` → `"guides/intro"`) |

These perform pure path transformations — they do not check whether content files exist.

## Language picker

The `LanguagePicker` component renders a `<select>` dropdown for switching between locales. It appears automatically in the site header when two or more locales are configured.

### Parameters

| Parameter | Type | Default | Description |
|---|---|---|---|
| `Locales` | `IReadOnlyDictionary<string, LocaleConfig>?` | `null` | The locale map from `DocsConfig.Locales` |
| `CurrentLocaleKey` | `string` | `""` | Active locale key (e.g. `"root"`, `"fr"`) |
| `CurrentContentPath` | `string` | `"/"` | Content path without locale prefix, used to build switch URLs |
| `BasePath` | `string` | `""` | Site base path (e.g. `"/docs"`) |
| `Translations` | `UiTranslations` | `UiTranslations.Default` | UI translations for accessibility labels |

### Behaviour

- Renders nothing when fewer than 2 locales are configured
- Uses a native `<select>` with `onchange` navigation — no JavaScript framework needed
- Builds switch URLs using `LocalePathHelper.PrefixPath()` so the user lands on the equivalent page in the target locale
- The `aria-label` uses `Translations.LanguageSelectLabel` for accessibility

### Output

```html
<nav class="language-picker" aria-label="Select language">
  <select aria-label="Select language" onchange="window.location.href=this.value">
    <option value="/docs/getting-started" selected>English</option>
    <option value="/docs/fr/getting-started">Français</option>
    <option value="/docs/de/getting-started">Deutsch</option>
  </select>
</nav>
```
