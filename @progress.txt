**Story 13: Route pattern matching** ✅ COMPLETED
- RoutePattern: parsed pattern with segment analysis (IsDynamic, HasCatchAll, segment counts)
- RouteMatcher: matches URL path → RouteEntry + extracted parameters
  - Static segments: exact match (case-insensitive)
  - Dynamic segments: capture single path segment
  - Catch-all segments: capture zero or more remaining path segments
- RouteComparer: priority ordering (most specific first)
  - Non-catch-all routes before catch-all
  - More segments = higher priority (tried first, won't conflict with shorter patterns)
  - More static segments = higher priority within same count
  - Segment-by-segment: static > dynamic > catch-all
  - Alphabetical tie-breaker for determinism
- RouteMatchResult: matched entry + extracted parameters dictionary (case-insensitive keys)
- 32 tests covering matching, priority ordering, edge cases
