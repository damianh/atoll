## Slice 2: Locale Configuration — COMPLETED

### Date: 2026-04-04

### Summary
Completed all 6 tasks in Slice 2 of the Lagoon i18n plan. Added locale configuration types, locale resolution from URL paths, and wired locale into `DocsLayout` for dynamic `lang`/`dir` attributes and per-locale translations.

### What was done

**New files:**
- `src/Atoll.Lagoon/I18n/LocaleConfig.cs` — sealed record with `Label`, `Lang` (required), `Dir` (default "ltr"), `Translations` (default `UiTranslations.Default`)
- `src/Atoll.Lagoon/I18n/LocaleResolver.cs` — static utility with `ResolvedLocale` record and `Resolve()` method. Handles: basePath stripping, locale prefix matching, root locale, case-insensitive matching, fallback to first locale, overload without basePath
- `tests/Atoll.Lagoon.Tests/I18n/LocaleResolverTests.cs` — 16 comprehensive tests covering null/empty locales, root locale, prefixed locale, RTL, BCP-47 hyphens, basePath, case-insensitive, fallback

**Modified source files:**
- `Configuration/DocsConfig.cs` — Added `Locales` (`IReadOnlyDictionary<string, LocaleConfig>?`, default null) and `DefaultLang` (string, default "en")
- `Layouts/DocsLayout.cs` — Added `CurrentPath` parameter; resolves locale via `LocaleResolver.Resolve()` at start of render; uses effective lang/dir/translations; always emits `dir` attribute on `<html>`

**Modified test files:**
- `Layouts/DocsLayoutTests.cs` — Updated `RenderLayoutAsync` helper to accept `currentPath`; updated assertion for `dir="ltr"` attribute; added 6 locale integration tests (default, French, RTL Arabic, root locale, per-locale translations, basePath)
- `Configuration/DocsConfigTests.cs` — Added 4 tests: null locales default, locales configuration, custom translations, RTL locale, custom DefaultLang

### Discoveries
- Adding `dir="ltr"` to `<html>` is always rendered (even for single-language sites). This is valid HTML and helps assistive technologies. It required updating 1 existing test assertion.
- `LocaleResolver.Resolve()` needs an overload without `basePath` to satisfy the coding standard of method overloads instead of optional parameters.

- `dotnet test tests/Atoll.Lagoon.Tests/` — 253 tests passed (224 existing + 29 new)
- Backward compatible: single-language sites render `lang="en" dir="ltr"` (same content, valid HTML)

## Slice 4: Language Picker Component — COMPLETED

### Date: 2026-04-04

### Summary
Completed all 4 tasks (32-35) in Slice 4 of the Lagoon i18n plan. Created the `LanguagePicker` component, integrated it into the `DocsLayout` header, added CSS styles, and wrote comprehensive tests.

### What was done

**New files:**
- `src/Atoll.Lagoon/Components/LanguagePicker.cs` — sealed `AtollComponent` that renders a `<nav class="language-picker">` containing a `<select>` with locale options. Parameters: `Locales`, `CurrentLocaleKey`, `CurrentContentPath`, `BasePath`, `Translations`. Renders nothing when `Locales` is null or has ≤1 entry. Uses `LocalePathHelper.PrefixPath` for URL building. Root locale key ("root") maps to empty prefix. Current locale option gets `selected` attribute. Labels are HTML-encoded via `WriteText`.
- `tests/Atoll.Lagoon.Tests/Components/LanguagePickerTests.cs` — 11 tests covering: null locales, single locale, multiple locales rendering, onchange handler, current locale selected, root locale selected, base path URL building, root locale URL building, translated aria-label, default aria-label, HTML encoding of labels.

**Modified source files:**
- `Layouts/DocsLayout.cs` — Added LanguagePicker rendering in header actions area (between social links and theme toggle), conditionally when `Config.Locales` has 2+ entries and locale is resolved. Passes `locale.Key`, `locale.ContentPath`, `Config.BasePath`, and resolved `translations`.
- `Styles/DocsTheme.cs` — Added `LanguagePickerStyles` const with CSS for `.language-picker` (inline-flex) and `.language-picker select` (custom appearance with dropdown chevron SVG, consistent styling with header elements, hover/focus states). Added to `[Styles(...)]` attribute.

### Design decisions
- Placed the LanguagePicker between social links and theme toggle in the header — natural location that groups all utility actions together.
- Used a native `<select>` with `onchange="window.location.href=this.value"` for simplicity and accessibility — no need for a client-side island.
- CSS uses `appearance: none` with an inline SVG chevron for consistent cross-browser styling while maintaining semantic HTML.
- The component requires locale to be resolved (`locale is not null`) before rendering, which naturally handles the single-language fallback.

- `dotnet test tests/Atoll.Lagoon.Tests/` — 303 tests passed (292 existing + 11 new)
- Backward compatible: single-language sites render no picker

## Slice 5: Content Organization (Folder-per-Locale) — COMPLETED

### Date: 2026-04-04

### Summary
Completed all 6 tasks (36-41) in Slice 5 of the Lagoon i18n plan. Created `LocaleContentResolver` for locale-aware content paths, extended `SidebarBuilder` with locale-filtered auto-generation, added per-locale search index generation, and implemented untranslated content notice rendering.

### What was done

**New files:**
- `src/Atoll.Lagoon/I18n/LocaleContentResolver.cs` — `ResolvedContent` record + `LocaleContentResolver` static class with `GetLocaleContentPath` and `StripLocaleFromSlug` methods. Handles folder-per-locale content paths, case-insensitive locale matching, and backslash normalization.
- `tests/Atoll.Lagoon.Tests/I18n/LocaleContentResolverTests.cs` — 15 tests covering locale path resolution, root locale, empty locale, backslashes, leading slashes, slug stripping, case-insensitive matching, partial match, hyphenated locales, edge cases.

**Modified source files:**
- `Navigation/SidebarBuilder.cs` — Added 4-arg `Build(currentPath, localePrefix, basePath, localeKey)` overload for locale-filtered auto-generation. Updated `SlugMatchesDirectory` to strip locale prefix from slugs before directory matching. Threaded `localeKey` through all internal `ResolveItems`/`ResolveItem`/`ResolveAutoGeneratedGroup` methods.
- `Search/LagoonSearchIndexGenerator.cs` — Added `GenerateAsync(documents, localePrefix)` overload that writes search index to `{localePrefix}/search-index.json`. Extracted `GenerateFromDocumentsAsync` and `ResolveOutputDirectory` private helpers.
- `I18n/UiTranslations.cs` — Added `UntranslatedContentNotice` property (default: "This page has not yet been translated.").
- `Layouts/DocsLayout.cs` — Added `IsUntranslatedContent` parameter. When true, renders a `<div class="untranslated-notice">` with the notice text from translations.
- `Styles/DocsTheme.cs` — Added `UntranslatedNoticeStyles` CSS constant with yellow/amber info-bar styling for `.untranslated-notice`.

**Modified test files:**
- `Navigation/SidebarBuilderTests.cs` — Added 8 locale-filtering tests: filter to locale key, root locale, no filtering with empty key, prefix auto-generated hrefs, mark group active, not include other locales, empty auto-generate dir, mark current page correctly.
- `Layouts/DocsLayoutTests.cs` — Added 4 untranslated notice tests: not rendered by default, rendered when IsUntranslatedContent=true, uses custom translations, contains correct CSS class.
- `Search/LagoonSearchIndexGeneratorTests.cs` — Added 4 per-locale search index tests: locale subdirectory, empty locale, root output, default overload.

### Key discoveries
- **PrefixHref double-prefixing**: Entry Hrefs passed to `SidebarBuilder` must be base hrefs (without locale prefix) because `PrefixHref` adds the locale prefix. Tests that provided already-locale-prefixed hrefs caused double-prefixing (e.g., `/docs/fr/fr/guides/start`).
- **SlugMatchesDirectory locale-aware**: The `localeKey` parameter enables stripping the locale prefix from entry slugs before matching against the directory string, so `fr/guides/start` with localeKey `fr` matches directory `guides`.

- `dotnet test tests/Atoll.Lagoon.Tests/` — 337 tests passed (303 existing + 34 new)
- Backward compatible: existing overloads work unchanged, new features are opt-in
