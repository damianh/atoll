### 2026-04-02 ‚Äî Story 15: API endpoint support

**Completed by**: claude-opus-4.6
**Commit**: (pending) (branch: astro-fork)

**What was done**:
- Created `IAtollEndpoint` interface in `src/Atoll.Routing/IAtollEndpoint.cs` ‚Äî uses C# default interface methods for GET/POST/PUT/DELETE/PATCH/HEAD/OPTIONS, each returning `Task<AtollResponse>?` (null = not implemented)
- Created `EndpointRequest` in `src/Atoll.Routing/EndpointRequest.cs` ‚Äî HTTP request abstraction (method, URL, headers, body stream) decoupled from ASP.NET Core
- Created `EndpointContext` in `src/Atoll.Routing/EndpointContext.cs` ‚Äî API context with route parameters, request, locals dictionary, typed accessors (`GetParameter`, `GetLocal<T>`)
- Created `AtollResponse` in `src/Atoll.Routing/AtollResponse.cs` ‚Äî HTTP response wrapper with factory methods: `Json<T>()`, `Text()`, `Html()`, `Redirect()`, `Empty()`, `MethodNotAllowed()`, `NotFound()`; JSON uses camelCase via `System.Text.Json`
- Created `EndpointDispatcher` in `src/Atoll.Routing/EndpointDispatcher.cs` ‚Äî static dispatcher that routes HTTP method to endpoint handler; uses `GetInterfaceMap` reflection to detect supported methods without invoking handlers; returns 405 with Allow header for unsupported methods
- Updated `RouteDiscovery.cs` ‚Äî added `IsRoutableType()` helper to scan for `IAtollEndpoint` types alongside `IAtollComponent`
- Created `tests/Atoll.Routing.Tests/EndpointTests.cs` with 66 tests covering:
  - EndpointRequest construction, defaults, null validation
  - EndpointContext parameter/local access, typed locals, missing key behavior
  - AtollResponse factory methods (JSON, text, HTML, redirect, empty, 405, 404)
  - JSON camelCase serialization
  - EndpointDispatcher routing all 7 HTTP methods to correct handlers
  - 405 response with Allow header for unsupported methods
  - GetSupportedMethods reflection for partial implementations
  - Route discovery integration scanning IAtollEndpoint types
  - End-to-end route match ‚Üí dispatch ‚Üí response flow
  - Middleware locals access pattern
